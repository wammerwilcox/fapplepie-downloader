# Fapplepie Downloader

[![CI](https://github.com/wammerwilcox/fapplepie-downloader/actions/workflows/ci.yml/badge.svg)](https://github.com/wammerwilcox/fapplepie-downloader/actions/workflows/ci.yml)
[![Image version](https://img.shields.io/github/v/tag/wammerwilcox/fapplepie-downloader?label=image&sort=semver)](https://github.com/wammerwilcox/fapplepie-downloader/pkgs/container/fapplepie-downloader)

Automated scraper and downloader for videos from [fapplepie.com](https://fapplepie.com).

## Features

- Multi-page scraping across the video directory
- Redirect following to resolve final hosting URLs
- Smart cache for previously resolved and downloaded URLs
- Fast downloads with `yt-dlp` and `aria2c`
- Daily scheduling through cron or Docker
- Optional outbound proxy support
- Docker and Docker Compose support

## Quick Start

### Standalone Python

```bash
python3 -m venv venv
source venv/bin/activate
pip install --require-hashes -r app/requirements.txt
python3 app/scraper.py --all
```

### Docker Compose

```bash
docker compose up -d
docker compose logs -f
docker compose down
```

The default compose file uses the pinned release image from GitHub Container Registry:
`ghcr.io/wammerwilcox/fapplepie-downloader:1.0.0`. Runtime state is written under `app/cache/`, `app/downloads/`, and `app/logs/`; those paths are ignored by Git.

For local image development, use:

```bash
docker compose -f docker-compose.dev.yml up -d --build
```

## Runtime Files

This public source tree intentionally does not include active runtime state.

- `app/video_urls.txt` is generated by scraping and used by download-only runs.
- `app/cache/processed_cache.json` is generated by the app and stores resolved/downloaded URL state.
- `app/video_urls.example.txt` and `app/processed_cache.example.json` are safe templates.

For a local download-only run, copy the URL template and add one URL per line:

```bash
cp app/video_urls.example.txt app/video_urls.txt
python3 app/scraper.py --download
```

## Usage

```bash
# Scrape only
python3 app/scraper.py --scrape

# Download from app/video_urls.txt
python3 app/scraper.py --download

# Scrape and download
python3 app/scraper.py --all

# Clear generated cache
python3 app/scraper.py --clear-cache

# Choose a download directory
python3 app/scraper.py --download --output-dir /path/to/videos
```

## Configuration

### Scheduling

Docker runs on a cron schedule by default:

```yaml
environment:
  CRON_SCHEDULE: "0 2 * * *"
```

For standalone cron setup, see [docs/CRONTAB_SETUP.md](docs/CRONTAB_SETUP.md).

### Optional Proxy

Set `NORDVPN_PROXY` to route requests through a SOCKS or HTTP proxy.

```yaml
environment:
  NORDVPN_PROXY: "socks5h://nl.socks.nordhold.net:1080"
  NORDVPN_USER: "${NORDVPN_USER}"
  NORDVPN_PASS: "${NORDVPN_PASS}"
```

Additional proxy controls:

- `NORDVPN_PROXY_SCOPE=fapplepie` proxies only fapplepie traffic.
- `NORDVPN_PROXY_SCOPE=all` proxies all scraper/downloader traffic.
- `SCRAPE_DIRECT_FALLBACK_ON_403=1` retries scraper requests directly after proxied HTTP 403 responses.

## Dependency Locking

Direct Python dependencies live in `app/requirements.in`; the installed lockfile is `app/requirements.txt`.

To refresh the lockfile:

```bash
python3 -m venv /tmp/fapplepie-lock-venv
/tmp/fapplepie-lock-venv/bin/python -m pip install --upgrade pip pip-tools
/tmp/fapplepie-lock-venv/bin/pip-compile --generate-hashes \
  --output-file app/requirements.txt app/requirements.in
```
